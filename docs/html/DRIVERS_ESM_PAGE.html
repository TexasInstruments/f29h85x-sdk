The ESM driver provides API to configure the ESM Registers. Below are the high level features supported by the ESM driver.

Features Supported

  • ESM CPU Instances, one for each CPU and a System ESM (SYS ESM) Instance.
  • Safety Aggregator Instance for Register Parity Error Aggregation for all ESM instances.
  • Supports upto 256 error event inputs fed to all ESM instances.
  • Error forcing capability
  • Selectable Low and High Priority Interrupt for each error event, and High Priority Watchdog Interrupt output for each ESM CPU instance.
  • Critical Priority interrupt output from all ESM instances.
  • Error Pin Output (level or PWM modes) from System ESM Instance to signal severe device failure.
  • Configurable timebase for error pin signal output.
  • Error Pin Monitoring.
  • Parity Detection and Commit/Lock for MMRs.

SysConfig Features

Note
It is strongly recommend to use SysConfig where it is available instead of using direct SW API calls. This will help simplify the SW application and also catch common mistakes early in the development cycle.
  • Error event selection and configuration.
  • Enabling and configuring ESM Low priority interrupt.
  • Enabling and configuring High priority watchdog.
  • Enabling and configuring NMI.
  • Error pin configuration.

Features NOT Supported

NA

Important Usage Guidelines

  • Group 0 events (Event Number 0 to 31) are mapped to High Priority Interrupt and High Priority Watchdog Counter is enabled by default.
