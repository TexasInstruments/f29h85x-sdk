CMPSS

The CMPSS library module is not yet available. However, this release does contain an example application that demonstrates using the INTERNAL_TEST feature to create a loopback between a CMPSS DAC (frequently referred to as "CompDAC") and the ADC. See the ADC "Internal Test Mode" section of your device technical reference manual for more details.

The SDL example configures an ADC channel to receive an input from a CMPSS DAC via the INTERNAL_TEST connection. The ADC is set up to use a software trigger, the trigger repeater block, and the Post-Processing Block (PPB) to accumlate ADC results. These accumlated conversion results can then be divided to calculate an average. The number of conversions being averaged can be adjusted in the accumulation_count field if more are required in case of noise. The test function runs a loop where the DAC value is steadily increased and the resulting ADC result is checked for correctness. The function also factors in allowed offset, gain, and INL error. The user may adjust these values based on their system noise, reference voltage, etc...

The function needs to be called twice for each CMPSS instance to test both the high and low CompDAC. These can be specified in the SDL_CMPSS_Config structure. This function not only performs a test of the CMPSS, but can also be used to test an ADC instance and supplement to ADC OSDETECT check which checks a channel connection but not channel conversion accuracy.

To perform this test, ADC VREFHI has to be the same voltage as the CMPSS reference (VDDA/VDAC, which is usually 3.3V). Syscfg has been configured to use InternalRef 3.3V, but the user can change it to ExternalRef if VREFHI is connected to 3.3V on the board. If VREFHI and VDDA/VDAC are not the same, then tweak allowedGainError value accordingly.
