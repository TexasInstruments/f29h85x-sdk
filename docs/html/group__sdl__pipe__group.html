int32_t SDL_PIPE_checkRegisterParity ( uint32_t errAggrBase,
uint32_t data,
uint8_t parity )

This API is used to validate the parity diagnostic mechanism for the given 32 bit data value and the mismatched parity. The hardware detects the parity mismatch and sets the corresponding error status in the Error Aggregator and ESM will generate an interrupt.

Parameters
errAggrBaseBase address of the Error Aggregator.
data32 bit data to be used for the parity check.
parityParity data which should not match the given data.
Returns
SDL_PASS - success
SDL_EFAIL - return a failure if it's unable to get interrupt
SDL_EBADARGS - API fails due to wrong base address
