Introduction

This page contains the user guide for the combined C29 Memory Allocation and SSU Configuration Tool. This tool is part of the F29x-SDK Sysconfig offering.

Usage of the tool is broadly divided into 2 profiles:

  • Memory Allocation : This profile is used only for memory partitioning between CPUs, i.e. creating input and output sections used in the linker command file. In this profile, SSU configurations are considered disabled
  • System Security Configuration : This profile is used for configuring various SSU settings related to memory protection, task isolation, debug zone separation, flash settings, boot configurations etc. For quick setup, a number of one-click options are provided.

Multi-Core Support

The SSU & Memory Allocation tool includes full support for multi-core SysConfig applications. The auto-allocator functionality manages Memory Regions across all enabled CPUs, included regions shared between multiple CPUs. For Memory Allocation profile, multi-core mode can be setup by simply adding memory regions in the required Sysconfig contexts. To use multi-core functionality in System Security configuration, enable the required cores in System Security module, and add the corresponding cores in the MULTI CORE tab in Sysconfig.

Summarized Views

These views are provided for a quick overview of the configured Memory Region and SSU settings. They can be accessed by opening the "Options" menu in Sysconfig. The view is specific to the currently active context on Sysconfig. To see the same view for another core, simply switch cores in the context window.

Options Menu

Memory Summary

This view provides a glance of all the created memory regions. Implicitly created memory regions (such as Peripheral Auto Regions created by System Security module) are also available here for inspection, which are not visible directly in the Memory Regions module. A total count of created regions is also provided to account for the same

SSU Summary

This view is applicable for System Security Configuration profile. It provides a summary of all the LINKs configured on the core, and the access permissions for the LINK's code memory regions to the data memory regions on the core.
