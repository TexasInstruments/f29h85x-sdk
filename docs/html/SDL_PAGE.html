Software Diagnostic Library

Introduction

The F29 Software Diagnostic Library (SDL) provides a set of APIs, examples, and tools to aid in the development of functional safety systems using F29x devices. The features provided in the library are based on the safety mechanisms specified in the supported devices' functional safety manuals. This support ranges from drivers to configure hardware safety mechanisms to software to implement tests of hardware features to software to implement software-based safety mechanisms (for instance, a CRC calculation). Note that the SDL itself aids in the detection of errors or enabling the hardware to detect them. The response to the reported errors is managed by the userâ€™s application based on their system requirements.

The source code of the SDL is provided for review and modification, although note that changing the code may impact your ability to leverage any certification that may exist for the library.

Attention
This is release is a beta version provided for evaluation of the library. As this is a beta release, only initial testing and reviews have been performed. Activities to bring the SDL project to full compliance with TI's functional safety quality software development process (such as software FMEA and dynamic analysis) are still in progress and no certification has yet been recieved.

The SDL consists of below sub-modules

  • ADC
  • CMPSS
  • DCC
  • Error Aggregator
  • ESM
  • LCM
  • MEMSS
  • PIPE
  • SIC
  • Stack Monitoring
  • SysCtl
