The AES driver provides API to configure and use the AES module within the DTHE (Digital Trusted Hardware Extension). Below are the high level features supported by the AES driver.

Features Supported

  • Efficient implementation of the Rijndael cipher (the AES algorithm) and a 128-bit polynomial multiplication
  • Support for key lengths: 128-bit, 192-bit, and 256-bit
  • Multiple operation modes:
    • Electronic Code Book (ECB)
    • Cipher Block Chaining (CBC)
    • Counter (CTR)
    • Cipher Feedback (CFB)
    • CMAC (Cipher-based Message Authentication Code)
  • Single-shot and multi-shot operation mechanisms
  • Two contexts: Public and Secure (Access Protection Scheme)

SysConfig Features

Note
It is strongly recommend to use SysConfig where it is available instead of using direct SW API calls. This will help simplify the SW application and also catch common mistakes early in the development cycle.
  • Configuration of AES operation modes (ECB, CBC, CTR, CFB, CMAC)
  • Key length selection (128, 192, 256 bit)
  • Initialization vector (IV) configuration
  • Context selection (Public or Secure)

Features NOT Supported

  • DMA functionality is not yet added in DTHE R5F, it will be added in the future.

Important Usage Guidelines

  • For multi-shot operations, the stream state must be properly managed (init, update, finish)
  • To cancel an existing stream, DTHE_AES_close should be called followed by DTHE_AES_open

Operation Modes

AES-ECB

AES ECB feedback mode

The Electronic Code Book (ECB) feedback mode of operation passes the input data directly to the basic crypto core, and the output of the crypto core is passed directly to the output buffer. For decryption, the crypto core operates in reverse, thus, the decrypt datapath is used for the data processing, where encryption uses the encrypt datapath.

AES-CBC

AES CBC feedback mode

The Cipher Block Chaining (CBC) feedback mode of operation XORs the input data with the initialization vector (IV) before it is passed to the basic crypto core. The output of the crypto core passes directly to the output buffer and becomes the next IV. For decryption, the operation is reversed, resulting in an XOR at the output of the crypto core. The input cipher text of the current operation is the IV for the next operation.

AES-CTR

AES CTR mode

The Counter (CTR) mode of operation encrypts the IV. The output of the crypto core (encrypted IV) is XORed with the data, creating the output result. The IV is built out of two components, one fixed part and a counter part. The counter part is incremented with each block. The counter width is variable per context and can be 16, 32, 64, 96 or 128-bit wide. Note that in this mode, encryption and decryption use the same operation.

AES-CFB

AES CFB mode

The full block (128-bit) Cipher Feedback (CFB) mode of operation for both encryption and decryption uses the IV as input for the crypto core; the result is XORed with the data. The result is fed back via the IV register, as the next input for the crypto core. The decrypt operation is reversed, but the crypto core is still performing an encryption.

AES-CMAC

AES CMAC mode

The One Key Mac mode of operation XORs the input to the crypto core with the IV. The crypto core output is then fed back as IV for the next block. The last data input block is XORed with an additional input value stored in the Temporary Buffer; this can be any precalculated value and is dependent on the alignment of the last input block. For CMAC mode, the tweak value (in temporary buffer) is calculated via K1 and K2 which is derived from Key value from software.

Example Usage

Single Shot Mechanism

This sequence performs encryption and decryption operations for AES-ECB/CBC/CTR/CMAC algorithms:

// Include the below file to access the APIs
#include <dthe.h>
// Function to open DTHE instance, enable DTHE engine
DTHE_Handle dtheHandle = DTHE_open();
// Function to open DTHE AES instance using the handle returned from DTHE_open()
DTHE_AES_Handle aesHandle = DTHE_AES_open(dtheHandle);
// Function to execute the AES driver with specified parameters
DTHE_AES_execute(aesHandle, params);
// Function to close DTHE AES driver
DTHE_AES_close(aesHandle);
// Function to De-initialize the DTHE instance
DTHE_close(dtheHandle);

Multi Shot Mechanism

This sequence performs encryption and decryption operations for AES-ECB/CBC/CTR/CMAC algorithms:

// Include the below file to access the APIs
#include <dthe.h>
// Function to open DTHE instance, enable DTHE engine
DTHE_Handle dtheHandle = DTHE_open();
// Function to open DTHE AES instance using the handle returned from DTHE_open()
DTHE_AES_Handle aesHandle = DTHE_AES_open(dtheHandle);
// Function to execute the AES driver with specified parameters and streamstate as init
DTHE_AES_execute(aesHandle, params_init);
// Function to execute the AES driver with specified parameters and streamstate as update
DTHE_AES_execute(aesHandle, params_update);
// Function to execute the AES driver with specified parameters and streamstate as finish
DTHE_AES_execute(aesHandle, params_finish);
// Function to close DTHE AES driver
DTHE_AES_close(aesHandle);
// Function to De-initialize the DTHE instance
DTHE_close(dtheHandle);

Canceling an Existing Stream

To cancel an existing stream, DTHE_AES_close should be called followed by DTHE_AES_open:

// Include the below file to access the APIs
#include <dthe.h>
// Function to open DTHE instance, enable DTHE engine
DTHE_Handle dtheHandle = DTHE_open();
// Function to open DTHE AES instance using the handle returned from DTHE_open()
DTHE_AES_Handle aesHandle = DTHE_AES_open(dtheHandle);
// Function to execute the AES driver with specified parameters and streamstate as init
DTHE_AES_execute(aesHandle, params_init);
// Function to execute the AES driver with specified parameters and streamstate as update
DTHE_AES_execute(aesHandle, params_update);
// Function to close DTHE AES driver
DTHE_AES_close(aesHandle);
// Function to open DTHE AES instance using the handle returned from DTHE_open()
aesHandle = DTHE_AES_open(dtheHandle);
// Start a new stream
// Function to execute the AES driver with specified parameters and streamstate as init
DTHE_AES_execute(aesHandle, params_init);
// Function to execute the AES driver with specified parameters and streamstate as update
DTHE_AES_execute(aesHandle, params_update);
// Function to execute the AES driver with specified parameters and streamstate as finish
DTHE_AES_execute(aesHandle, params_finish);
// Function to close DTHE AES driver
DTHE_AES_close(aesHandle);
// Function to De-initialize the DTHE instance
DTHE_close(dtheHandle);
