Introduction

This example configures CPU Timer0, 1, and 2 to be profiled using the ERAD module. Included is a JavaScript file, erad_ex1_profile_interrupts.js, which is used with the scripting console to program ERAD registers and view profiling data.

The example does the below

  • Configures CPU Timer 0, 1, and 2.
  • Only CPU timer 2 ISR is profiled in this example.
  • BUSCOMP4 and BUSCOMP5 are configured to monitor the start address and end address of the CPUTIMER2 ISR respectively.
  • COUNTER2 is configured in start-stop mode with start event as BUSCOMP4 and stop event as BUSCOMP5.
  • COUNTER0 is configured in rising-edge count mode with counting input as system event TIMER2_INT.
  • COUNTER1 is configured in start-stop mode with start event as TIMER2_INT and stop event as BUSCOMP4.
  • The included JavaScript file, erad_ex1_profile_interrupts.js, configures these ERAD modules and gets the profiling data.

Steps to Run the Example

  • When using CCS projects to build, import the CCS project for the required combination and build it using the CCS project menu (see CCS Build Project).
  • When using makefiles to build, note the required combination and build using make command (see Makefile based Build)
  • Establish connections as mentioned in External Connections section.
  • Launch a CCS debug session and run the executable, see Build, Load and Run
  • To run the ERAD script, run the following command in the scripting console: require("<proj_workspace_path>\\erad_ex1_profile_interrupts\\erad_ex1_profile_interrupts.js")
  • Note that the script must be run after loading and running the .out on the C29x core.
  • View the profiling results in the console log.
