Introduction

This example demonstrates how to create and apply Security settings using SSU module. It creates multiple LINKS and associated APRs and Stacks in SSUMODE 2. The code execution switches between LINKs in a secure manner. The memory regions are created and are protected using APR configurations

Watch Variables

  • adc_int_count - The ADC isr count make sure it is incrementing continously
  • cpuTimer0IntCount - CPU timer isr count make sure it is incrementing continously
  • cpuTimer1IntCount - CPU timer isr count make sure it is incrementing continously
  • cpuTimer2IntCount - CPU timer isr count make sure it is incrementing continously

External Connections

  • None

Steps to Run the Example

  • When using CCS projects to build, import the CCS project for the required combination and build it using the CCS project menu (see CCS Build Project).
  • FLASH_HSFS and FLASH_HSSE build configurations are supported
    • FLASH_HSFS - Used in HSFS device type
      • The example.out is programmed using flash plug in
    • FLASH_HSSE - Used in HSSE device type
      • Creates 'seccfg.bin' file which should be flashed using HSM before the example binary, refer TIFS documentation for more details
      • Creates certificate appended example binary (example_cert.bin) which should be flashed using HSM, refer TIFS documentation for more details
  • Establish connections as mentioned in External Connections section.
  • Launch a CCS debug session and run the executable, see Build, Load and Run
